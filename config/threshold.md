# LightFlow 复杂度阈值配置

此文件控制任务何时触发 Brainstorm 和 Plan 流程。
低于阈值的任务将直接进入编码阶段。

## 阈值设置

```yaml
# 文件数量阈值
# 改动文件数超过此值 → 触发完整流程
max_files_changed: 3

# 代码行数阈值
# 新增/修改代码行数超过此值 → 触发完整流程
max_lines_changed: 100

# 风险等级
# low    — 不触发流程（纯 UI 调整、文案修改）
# medium — 按阈值判断（默认）
# high   — 强制触发完整流程
risk_level: medium

# 是否在变更涉及以下内容时强制触发完整流程
force_full_flow_on:
  - database_schema   # 数据库结构变更
  - api_redesign      # API 重设计
  - cross_module      # 跨模块改动
  - third_party       # 引入新依赖/第三方服务
```

## 手动覆盖

在对话中使用以下指令：

```
# 强制走完整流程（覆盖阈值）
"用完整流程处理这个任务"

# 跳过所有流程，直接改
"直接改，不用问"
"跳过 Plan 和 Review"

# 临时调整阈值
"这次当复杂任务处理"
"这只是个简单改动"
```

## 注意事项

- 阈值不是硬限制，Agent 可以根据需求复杂性自行判断
- 如果任务看似简单但涉及关键逻辑（如支付、权限），应升格处理
- 阈值可以随着项目成长而调整——新项目需要更多流程，成熟项目可以更自由
