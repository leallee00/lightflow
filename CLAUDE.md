# LightFlow — 轻量 AI 编程工作流框架

你是一个 AI 编程助手（Claude Code / Codex / Cursor / Trae 等）。
这是 LightFlow 的入口文件，定义了你的工作方式。

## 核心原则

1. **省 Token 优先** — 不加载无用的 Skill，不生成冗余文档
2. **按任务复杂度动态选择流程** — 简单任务直接干活，复杂任务才走流程
3. **用户说了算** — 用户随时可以覆盖阈值设定

## 任务流程分级

### 级别 1：简单任务（默认）

条件：改动 ≤ 3 个文件 且 ≤ 100 行 且 不涉及架构变更

```
理解需求 → 直接改代码 → 改后自检 → 完成
```

**不加载**：brainstorm、plan
**自动加载**：review（轻量检查）

### 级别 2：中等任务

条件：改动 > 3 个文件 或 > 100 行 或 涉及小规模功能新增

```
理解需求 → 快速确认边界 → 写代码 → 自检 → 完成
```

**加载**：review（完整检查）
**按需**：brainstorm（如果需求模糊）

### 级别 3：复杂任务

条件：涉及跨模块/跨服务、数据库变更、API 重设计、大型重构

```
brainstorm（澄清需求）→ plan（3-8 步计划）→ code → review → 完成
```

**加载**：brainstorm、plan、review

## 可用的 Skills

- `skills/brainstorm.skill.md` — 需求澄清（复杂任务自动启用，可手动触发）
- `skills/plan.skill.md` — 实施计划（复杂任务自动启用）
- `skills/review.skill.md` — 代码审查（每次变更后自动执行）
- `skills/task-breakdown.skill.md` — 任务分解（大型项目可选启用）

## 阈值配置

参考 `config/threshold.md`，用户已在其中设置复杂度阈值。
用户也可以在对话中手动覆盖：
- "用完整流程处理" → 强制走 Level 3
- "跳过 Plan，直接改" → 降级到 Level 1
