# Task Breakdown — 任务分解 Skill

## 作用

将大型重构或新功能开发拆解为可执行的子任务序列。
适用于不适合一口吃掉的复杂工作。

## 触发条件

- 用户主动要求分解任务
- 任务评估为大型重构（影响 5+ 个文件或涉及核心逻辑重写）
- 项目初始化/大功能从零搭建

## 工作流程

```
1. 理解整体目标
   └─ 确认最终交付物是什么

2. 拆分子任务
   └─ 每个子任务独立可交付（能单独测试/验证）
   └─ 标注依赖关系（A → B → C）

3. 确定优先级
   └─ P0：核心功能，必须先做
   └─ P1：重要功能
   └─ P2：锦上添花

4. 输出任务列表
   └─ 纯文字，不生成文件
   └─ 5-15 个子任务
```

## 输出格式

```
## 任务分解：XXX

### P0（必须先做）
1. [模块] 做什么 — 预计工作量：Xh
2. [模块] 做什么 — 预计工作量：Xh
   └─ 依赖：任务 1

### P1（重要）
3. ...

### 总览
共 X 个子任务，预计 Y 小时
```

## 约束

- 子任务不超过 15 个，超过需要合并
- 每个子任务标注预计工作量（小时）
- 标注依赖关系
- 不创建文件/Git Worktree
